How it works

From a one-line ticket to a spec, a scaffold, and a sprint

PageSpace isn't a chatbot in a side panel. It's a workspace where AI agents do the research, drafting, and planning inside your pages, using real tools.

1

Drop the ticket into a drive

Paste the feature request, the bug, or the design question into a page. The agent reads the architecture docs and existing code pages already in the drive for context, then drafts an RFC straight into a Document.

create_pageread_pagemulti_drive_search
2

The agent researches the unknowns

It runs live web searches for the library, the API, or the failure mode, fetches the docs as clean markdown, and searches your own drives by glob or regex to find every call site — then writes the findings into the RFC with links.

web_searchweb_fetchglob_searchregex_search
3

Scaffold and break down the work

The agent writes a starter implementation into a Code page (Monaco editor), makes precise line-level edits with replace_lines, and breaks the RFC into ordered tasks on a Task List board — with your team's own statuses.

replace_linescreate_taskcreate_task_statusreorder_task
4

Review against a second agent

Spin up an AI Chat agent, brief it as your reviewer, and consult it with ask_agent. It red-teams the design and the code; the team hashes it out in a Channel — all in the same drive.

update_agent_configask_agentsend_channel_message
The toolkit

Code pages, real editor

A full Code page type on the Monaco editor. Agents scaffold implementations and make precise edits with replace_lines — your snippets and configs live in the drive, not a gist.

📄

RFCs in the editor

Agents draft design docs and runbooks straight into Document pages on the TipTap editor, with source links inline and clean line-level edits as the plan changes.

✅

Task boards agents fill

Task List pages with custom statuses. Agents create, reorder, and update tasks via create_task and reorder_task, so the board reflects the spec.

🔎

Search across everything

Find any call site or doc with glob_search, regex_search, and multi_drive_search — plus live web research with web_search and web_fetch.

Reviewer agents

Every AI Chat page is a configurable agent. Brief one as a reviewer or an on-call responder, then consult it with ask_agent — a second opinion that lives in the drive.

Channels + real-time

Channel pages for team back-and-forth, with agents posting via send_channel_message — and real-time collaboration so people and agents work the same page at once.

Plan ↔ execution

The spec and the board stay in sync.

When the agent drafts an RFC, it breaks the work into tasks on the same drive's board — with the statuses your team actually uses. Change the plan and the agent updates the board. No more spec that says one thing while the tickets say another.

✓

Tasks created from the spec

The agent turns each RFC section into an ordered task with create_task and reorder_task — traceable back to the doc.

✓

Your statuses, not ours

Define create_task_status columns that match your flow — Backlog, In review, Blocked, Done.

✓

You stay in control

Edit any task or doc live — refresh protection means the agent never clobbers the line you're working on.
