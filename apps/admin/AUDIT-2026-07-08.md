# Admin App Audit — 2026-07-08

Full audit of `apps/admin` (Next.js 15, port 3005) driving a complete revamp.
Method: four parallel deep audits (UI/UX + navigation, data correctness vs `packages/db` schema,
redundancy/dead code vs monorepo, admin-controls gap analysis), verified against source.

**Verdict: revamp required.** The app is a flat list of 11 disconnected dashboards with two
competing navigation systems, ~1,600 LOC copy-pasted from `apps/web`, ~455 LOC of dead code,
three pages reporting the same billing data three different ways, silent error states that render
as "all zeros," no way to sign out, and almost no admin *controls* — while fully-built backends
(GDPR export/erase, suspend, session revocation, role change) sit unwired.

---

## 1. Navigation & Information Architecture (incoherent)

- **No coherent home.** `/` redirects to `/monitoring` (`src/app/page.tsx:4`), but the first tab
  "Overview" points to `/dashboard` (`AdminLayoutClient.tsx:36`). The landing page and the
  highlighted tab disagree.
- **Two competing nav systems.** The 11-tab flat bar (`AdminLayoutClient.tsx:71-112`) plus a
  stale `/dashboard` card launcher that links only 6 of 11 sections (`dashboard/page.tsx:14-51`).
- **No grouping.** Tab order interleaves analytics, billing, system, and people surfaces randomly.
  11 flat top-level tabs is past the point where a flat bar works.
- **No logout, no account menu, no breadcrumbs, no global search, no keyboard nav.** There is
  literally no way to sign out of the admin console.
- **Fragile active-tab detection**: hand-rolled `pathname.includes()` ladder; any new route
  silently highlights the "Users" tab (`AdminLayoutClient.tsx:36-45`).
- Inner tab state (Monitoring, Global Prompt) is not URL-driven — lost on refresh, not deep-linkable.

## 2. Mobile (not usable)

- Tab bar `flex-wrap` + `overflow-x-auto` conflict: wraps to 3-4 rows on phones, consuming the
  viewport (`AdminLayoutClient.tsx:72`).
- `UserActivityHeatmap` renders a ~430px fixed-width grid with **no overflow wrapper**
  (`monitoring/components/UserActivityHeatmap.tsx:55-78`) — overflows every phone.
- Users row header packs 5 badges with no `flex-wrap` (`UsersTable.tsx:267-305`) — crushed on mobile.
- Monitoring header (title + select + 3 icon buttons) never wraps (`monitoring/page.tsx:99-148`).
- Icon/sm ghost buttons below 44px touch targets throughout.

## 3. Redundancy & duplication

**Within the app (user-visible):**
- **Unit Economics ≈ AI Billing (~80% same page).** Identical Range/Granularity/CSV controls,
  margin-by-model/provider, margin-over-time, per-user spend tables; helpers (`usd`, `pct`,
  `marginClass`, `userLabel`, `formatPeriodLabel`) copy-pasted verbatim in both files.
- **"AI cost" reported 3 ways** (Monitoring quick-stat + AiUsageBreakdown, Unit Economics,
  AI Billing) with three different definitions and no cross-linking.
- **"Top spenders" appears 3 times** with 3 schemas (by $, by charged credits, by tokens).
- **Tier breakdown duplicated** (Growth pie vs AI Billing table).
- **Four different stat-card idioms** for the same concept; hardcoded `text-blue-600`-style
  colors instead of theme tokens in the users/audit-logs/support/tables overview rows.

**Against the monorepo (dev-facing, ~1,630 LOC):**
- Entire `src/lib/onboarding/` tree (~1,039 LOC incl. duplicate test) is byte-identical to
  `apps/web/src/lib/onboarding/*` (only `drive-setup.ts` differs by a 2-line import).
- `src/lib/auth/admin-role.ts` (88) and `csrf-validation.ts` (141): byte-identical to web.
- Stripe helpers (~190 LOC: `stripe-config.ts`, `stripe-customer.ts`, `stripe-errors.ts`,
  `stripe/client.ts`) byte-identical to web; `cookie-config.ts` differs in 8 lines.
- → belongs in `packages/lib` (onboarding, auth, stripe), imported by both apps.

**Dead code (~455 LOC, safe to delete now):**
- `src/components/ui/alert-dialog.tsx` (157), `dialog.tsx` (143), `tooltip.tsx` (61): zero importers.
- Deprecated 410-Gone `PUT /api/admin/users/[userId]/subscription` route (29 LOC), no callers.
- `getPerformanceMetrics` + dead `performance`/`api-metrics` cases in
  `api/monitoring/[metric]/route.ts` (~65 LOC): frontend never fetches them.
- Unused deps: `@radix-ui/react-dropdown-menu` (fully unused), `react-dialog`,
  `react-alert-dialog`, `react-tooltip` (only used by dead components).
- Unused local `deployment-mode.ts` exports (`isTenantMode`, `isCloud`).
- Two entry points for `cn` (`@/lib/utils` vs `@/lib/utils/index`), imports split 7/10.

## 4. UX quality defects

- **Silent failure (HIGH):** Growth catches fetch errors with `catch {}` and no error UI
  (`growth/page.tsx:104-106`); Monitoring logs to console and renders **zeros** for everything
  (`monitoring/page.tsx:52-54`) — a dead API is indistinguishable from a healthy-but-idle system,
  on the monitoring page of all places.
- **No pagination on Users (HIGH):** fetches ALL users, filters/sorts client-side
  (`users/page.tsx:72-79`, `UsersTable.tsx:127-138`). Freezes at scale.
- **Stripe link hardcoded to test mode (HIGH):** `dashboard.stripe.com/test/customers/…`
  (`UsersTable.tsx:512`) — wrong dashboard in production.
- Hardcoded gift-tier dollar values in UI (`UsersTable.tsx:442-444`) drift from Stripe prices.
- Native `alert()`/`confirm()` for gift/revoke money flows; leftover `console.log`s.
- Growth manual refresh flashes the page back to skeletons despite "keep stale data" intent
  (`growth/page.tsx:99`).
- Inconsistent loading idioms (skeleton cards / inline skeletons / spinner rings / raw pulse);
  Support renders literal `0`s while loading (`support/page.tsx:55,155`).
- Inconsistent filter semantics on Audit Logs (dropdowns auto-apply; search needs Enter/button;
  "Apply Filters" button is redundant for dropdowns).
- No date-range control on Growth/Compaction while Monitoring/Unit-Econ/AI-Billing have one.
- Export inconsistency: Monitoring dumps client-side JSON of in-memory state; others use server CSV.
- Global Prompt mode label: anything ≠ `fullAccess` renders as "Read Only" (`global-prompt/page.tsx:300`).
- Login page bypasses the design system with hand-rolled inputs (`login/page.tsx:89-108`).
- Nav alert badges fetched once, never refreshed; same `billingAlert` dot glued to two tabs
  (`AdminLayoutClient.tsx:49-57`).
- Every content page is `"use client"` + `useEffect` fetching — zero server rendering, every
  navigation flashes skeletons.

## 5. Data correctness (audited against `packages/db` schema + `packages/lib` services)

**Verified correct:** core cost math uses real `aiUsageLogs.cost` (not stale AI_PRICING tables);
1:1 ledger↔usage join is dup-safe; UTC date-bucketing is right; rolling 30/60-day MoM avoids
partial-month bias; schema browser exposes metadata only; the 3b89613b7 PII-decrypt fix holds for
users/audit-logs/monitoring; no fake/random data anywhere.

**HIGH:**
- **H1 — Ciphertext PII leaks via gift-subscription route.**
  `api/admin/users/[userId]/gift-subscription/route.ts:99,123,135,205,212` loads the target user
  without `decryptUserRows`, then writes `Gift for ${email}` (ciphertext) into the **Stripe coupon
  name**, returns ciphertext in the success message shown to admins, and `maskEmail`s ciphertext
  into audit metadata. Same bug class the recent decrypt fix targeted; this route was missed.
- **H2 — Paying subscribers on legacy prices counted as "free".**
  `monitoring-queries.ts:1124` calls `getTierFromPrice(r.stripePriceId)` without the price amount,
  so the `LEGACY_PRICE_AMOUNTS` fallback never triggers; grandfathered subs bucket as `free` in
  the AI-Billing "Subscribers by tier" panel.

**MEDIUM:**
- **M1 — MAU/DAU headline vs trend use different data sources.** Headline = distinct
  `sessions.lastUsedAt` (`monitoring-queries.ts:1219-1236`); trend charts = distinct
  `activityLogs.userId` (`:1270-1315`). Rendered side by side on Growth; they cannot reconcile.
  Also a third "active" definition in `getSystemHealth` (15-min sessions).
- **M2 — Gifted subs inflate paid-tier counts.** `getActiveSubscriptionsByTier` (`:1117-1121`)
  has no `gifted = false` filter and excludes `trialing`, contradicting `payingUsers` (`:1263-1266`).
- **M3 — "Credit revenue" overstates revenue.** `getCreditRevenue` (`:1082-1115`) sums
  `topup_purchase` (real cash) + `monthly_grant` (allowance grants, incl. gifted/free users) and
  the AI-Billing headline renders the total as revenue. No true subscription MRR exists anywhere.
- **M4 — Compaction summary mixes windows and truncates.** 24h logs capped at `limit(50)` feed
  "total cost" placed next to 7d counts; `avgSummaryTokens` computed from latest 20 rows unbounded
  in time (`api/admin/compaction/route.ts`).

**LOW:**
- **L1** — Outstanding debt is all-time point-in-time but returned inside the range-scoped
  unit-economics summary (`monitoring-queries.ts:578-586`).
- **L2** — "Top Spenders" (`AiUsageBreakdown`) shows provider cost (what we paid), labeled as
  user spend (`monitoring-queries.ts:277-289`).
- **L3** — Alert thresholds are magic numbers (`errorRate > 5`, `holdCount > 50`) not tied to any
  configured SLO constant (`api/admin/alerts/route.ts:188-193`).
- **L4** — Failed-login analytics surfaces raw `systemLogs.metadata` (may contain plaintext email).

## 6. Admin controls gap (read-only console)

**Existing mutations (the complete list):** create user (on-prem only UI), gift/revoke
subscription, force-revoke paid subscription, mark contact resolved. Everything else is read-only.

**Built backends with NO route/UI (pure wiring gaps, XS effort):**
1. **Suspend/unsuspend user** — `users.suspendedAt/suspendedReason` enforced at login and session
   validation (`session-service.ts:104`), nothing sets them.
2. **Revoke sessions / force logout** — `sessionService.revokeAllUserSessions`,
   `revokeWebForUser`, `revokeAdminForUser` exist, no callers.
3. **GDPR erase** — `DELETE /api/admin/users/[userId]/data` fully built, no button.
4. **GDPR export** — `GET /api/admin/users/[userId]/export` fully built, no button.
5. **Change role (promote/demote admin)** — `updateUserRole()` with adminRoleVersion bump exists
   (`admin-role.ts:18`), no route calls it.

**Small new backend (S/M):** per-user credit balance + ledger view (tables + `getCreditBalance`
exist); manual credit adjustment (ledger supports `'adjustment'` entryType, no manual-adjust fn);
DSR queue/SLA dashboard (exists in apps/web admin GDPR routes, not surfaced here); resend/verify
email; on-demand retention runs; editable global prompt (currently GET-only end to end).

**Dangerous controls lacking guardrails:**
- "Force Revoke Subscription" cancels a **paying** customer's Stripe sub immediately, guarded by
  browser `confirm()`, no reason captured, generic audit row only (`UsersTable.tsx:518-545`).
- Create-user can mint a pre-verified **admin** account in one submit — privilege-escalation path
  with no distinct audit event.
- Gift subscription fires real Stripe mutations with no confirmation at all (`UsersTable.tsx:447`).
- Hash-chained audit log records method+path only; action parameters (tier, reason) live only in
  app logs — the tamper-evident trail can't say *what/why*.

**Posture (good bones):** session-cookie auth + CSRF on mutations (`withAdminAuth`), monotonic
`adminRoleVersion` invalidates demoted admins mid-session, tamper-evident hash-chained audit log
with integrity endpoint.

## Revamp direction

1. **New shell:** grouped sidebar (Overview / Analytics / Billing / People / System) + header with
   account menu (logout!), mobile drawer, URL-driven state, one coherent home.
2. **Consolidate:** merge Unit Economics + AI Billing (+ Monitoring's AI-usage tab) into one
   Billing surface; single definitions for AI cost / top spenders / tiers.
3. **Design system:** one StatCard, one PageHeader, one loading/error/empty-state pattern
   (errors must be loud, never zeros); theme tokens, no hardcoded palette.
4. **Controls:** wire suspend, session revocation, role change, GDPR export/erase with typed
   confirmations + mandatory reasons written into the audit trail; fix force-revoke guardrails.
5. **Correctness:** fix findings in §5; paginate Users server-side; env-aware Stripe links.
6. **Dedupe:** delete dead code + unused deps now; lift onboarding/auth/stripe duplicates to
   `packages/lib` (or schedule as immediate follow-up).
7. **Mobile pass:** responsive shell, overflow-safe charts/tables, wrapped headers, touch targets.
